开源音乐评估模型调研报告

外网可用的开源模型全景

1. 模型涉及的能力维度

音乐评分评估可以从以下几个维度拆解：

  • 音高/音准 (Pitch) — 检测演唱者的音高频率，与参考旋律对比
  • 节奏 (Rhythm/Tempo) — 检测节拍是否准确、时值是否稳定
  • 音色/音质 (Timbre) — 声音的明亮度、泛音结构等
  • 和弦/调性 (Tonality) — 和弦识别、调性检测
  • 音符转录 (Music Transcription) — 将音频转成 MIDI/乐谱
  • 活跃度/表现力 (Mood/Dynamics) — 演唱的情感表达和动态变化

2. 与本项目的交叉

针对我们本次的初级音乐考试（学生对着摄像头唱指定曲目，系统做音准+节奏评分），核心需要的是：

必须用的能力

  • ✅ 音高追中 (Pitch Tracking) — 核心
  • ✅ 节奏检测 (Beat/Tempo) — 核心
  • ✅ 人脸检测 (Face Detection) — 身份对应

暂时不用的能力

  • ❌ 和弦/调性分析 — 初级考试不需要
  • ❌ 情感识别 — 超出初级范围
  • ❌ 音符转录 — 我们的参考旋律已知
  • ❌ 音色评估 — 初级考试不评分

3. 推荐的开源模型清单

CREPE ⭐ 首选

MIT License 深度 CNN

能力：单声道音高追踪，输出频率 (Hz) + 置信度

定位：行业 SOTA 音高检测模型，ICASSP 2018 发表

用法：pip install crepe，Python 调用或命令行

在这个项目中的作用：检测学生演唱的每帧音高，与参考旋律对比 → 音准得分

Librosa

ISC License DSP 分析

能力：节拍检测 (tempo)、节拍追踪 (beat tracking)、频谱分析、MFCC

定位：业界标准音频分析工具包，广泛用于 MIR 研究

用法：pip install librosa

在这个项目中的作用：通过 onset detection + beat tracking 对比学生的节奏 → 节奏得分

Essentia

AGPL v3 综合性引擎

能力：Melody 提取、节拍检测 (TempoCNN)、调性检测、预训练音频嵌入

定位：上音/UPF 开发的全能音乐分析引擎，内置大量预训练模型

用法：pip install essentia

在这个项目中的作用：可作为 CREPE + Librosa 的备选或补充（内置 CREPE + TempoCNN）

Basic Pitch (Spotify)

Apache 2.0 多声部转录

能力：音频转 MIDI，支持多声部和多乐器

定位：Spotify 开发的轻量级音符转录

用法：pip install basic-pitch

在这个项目中的作用：可备选——但初级单声道演唱，CREPE 更直接

OpenCV (人脸检测)

Apache 2.0 计算机视觉

能力：人脸检测、人脸识别、摄像头调用

定位：业界标准计算机视觉库

用法：pip install opencv-python

在这个项目中的作用：调用摄像头 → 人脸捕捉 → 与考生身份对应

4. 模型组合建议

推荐组合（方案 A 的技术选型）：

  • CREPE — 音高追踪（核心评分引擎）
  • Librosa — 节奏分析（补充评分维度）
  • OpenCV — 摄像头调用 + 人脸检测

这个组合的优点是：

  • 三个库都是成熟的、活跃维护的开源项目
  • CREPE 在音高检测精度上是公认的标杆
  • Librosa 的节奏分析是业界标配
  • OpenCV 的人脸检测功能足够用于身份确认
  • 全部 pip install 即可使用，无复杂配置
