音乐考试评分系统 — 3 种技术方案

基于开源模型的实现路径对比

核心模型选择

调研后推荐的组合：

  • CREPE（MIT License）— 深度学习音高追踪，准确率行业领先
  • Librosa — 节奏/节拍分析，音色特征提取
  • 两者均为 Python 库，pip install 即可使用
A

React 前端 + Python 后端（推荐）

推荐
  • 前端：React + Webcam API 录唱 + 实时反馈界面
  • 后端：Python FastAPI + CREPE（音高）+ Librosa（节奏）
  • 评分：实际音高 vs 参考旋律对比 → 音准分 + 节奏分
  • 优点：界面专业、前后端分离、可扩展
  • 代价：需要同时维护前后端
B

Python 一体化应用（Streamlit/Gradio）

快速原型
  • 纯 Python：Streamlit 做界面 + CREPE + Librosa
  • Webcam 通过 OpenCV 调用
  • 优点：开发速度最快、单一代码库
  • 代价：界面较简陋、定制化困难
C

纯前端方案（TensorFlow.js + 浏览器推理）

探索性
  • TF.js 运行 CREPE 模型，音高检测在浏览器完成
  • 无需服务器，离线可用
  • 优点：部署简单、隐私性好
  • 代价：浏览器模型精度打折、节奏分析有限

我的推荐：方案 A — 界面专业、评分可靠、后续扩展空间大（加入更多题型、成绩管理等）。符合这个项目作为正式产品的定位。
