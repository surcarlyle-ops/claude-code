## 一、产品定义

### 1.1 产品名称建议、核心定位与愿景

**产品名称**：MusicExam —— 音乐考试模拟练习平台

**核心定位**：面向初高中生的 AI 音乐评分工具。学生对着屏幕唱指定曲目，系统通过 AI 模型做音准 + 节奏评分，实时反馈结果并给出改进建议。

**产品愿景**：让每个学生都能在家获得"跟老师一对一练歌"的体验，降低音乐自学门槛，成为最易用的音乐考试自评工具。

### 1.2 产品核心价值与竞品的关键差异

| 维度 | MusicExam | 传统方式（线下课/老师面批） | 竞品 App（如 Yousician） |
|------|-----------|--------------------------|------------------------|
| 使用门槛 | 浏览器打开即用，零安装 | 需约课、通勤 | 需下载 App，部分付费 |
| 反馈及时性 | 演唱中实时展示音准曲线 + 分项分数 | 需下课等老师点评 | 实时但多针对乐器 |
| 考试针对性 | 直接对应学校音乐考试评分维度 | 通用教学，非考试导向 | 乐器教学导向 |
| 防作弊 | 人脸拍照绑定 + 在位检测 | 老师现场监督 | 无身份绑定 |
| 成本 | 开源模型，零 API 费用 | 课时费高 | 订阅制付费 |

### 1.3 核心用户画像

**典型用户：李明（初二学生）**

- **年龄**：13岁，初中二年级
- **背景**：普通中学，音乐课是必修但有考试压力
- **技术能力**：会用手机看视频、会用电脑浏览器，无音乐基础
- **行为习惯**：
  - 放学回家后用家里电脑练习
  - 不太愿意让父母听到自己唱歌（容易害羞）
  - 考前一周集中练习，每次 10-20 分钟
  - 平时不看乐谱，考试时对着简谱哼唱
- **情感状态**：对唱歌没信心，怕跑调被同学笑，但考试不得不考
- **真实需求**：一个人偷偷练，知道自己唱得对不对，考前心里有底

**次要用户：王老师（初中音乐老师）**

- 需要检测全班学生练习进度
- 希望通过量化数据追踪学生进步
- MVP 阶段暂不覆盖教师端

### 1.4 核心使用场景

| 场景 | 用户诉求 | 产品提供的方案 |
|------|---------|--------------|
| 考前模拟练习 | "想知道考试曲目我唱得对不对" | 选曲页浏览曲目 + 简谱预览 + 试听参考旋律 |
| 自主练习纠正 | "唱跑调了但我自己听不出来" | 实时音高曲线对比（学生 vs 参考） |
| 节奏不稳 | "老是快/慢半拍" | 节奏分项反馈 + 错误小节标记 |
| 考试结果反馈 | "考完了想知道哪里有问题" | 总分 + 分项分 + 具体错误小节 + 改进建议 |
| 防作弊 | "怎么证明是我唱的" | 人脸拍照绑定 + 全程在位检测 |

### 1.5 行业现状问题、用户真实痛点拆解及产品对应解决方案

| 行业问题 | 用户真实痛点 | 产品解决方案 |
|---------|------------|------------|
| 学校音乐课时有限，老师无法逐个辅导 | 练歌没人听，不知道自己唱得对不对 | AI 实时评分，一个人也能练 |
| 现有音乐 App 以乐器教学为主 | 没有针对"唱歌考试"的工具 | 专注人声演唱评分，不教乐器 |
| 家长/学生不愿意为练习工具付费 | 需要免费或极低成本的方案 | 开源模型栈，零 API 成本 |
| 音乐考试"替考"存在 | 考试真实性难保证 | 人脸拍照绑定 + 在位检测 |
| 学生不好意思让别人听自己练歌 | 练歌时缺乏安全感 | 单人私密练习场景，无需他人陪同 |

### 1.6 核心页面与功能详情

该部分已在线下设计文档中有完整描述（`docs/specs/design-spec.md`），此处提炼重点：

**页面一：登录 / 考生身份确认** `/login`
- 人脸拍照绑定（姓名 + 学号 + 照片）
- 摄像头权限管理 + 异常状态处理
- **用户体验价值**：5 秒完成注册，零学习成本

**页面二：选择考试曲目** `/songs`
- 难度筛选（初/中/高级）
- 曲目卡片：简谱缩略图 + 试听 + 时长
- 确认弹窗：选择后二次确认
- **用户体验价值**：看得见考题内容，提前心里有数

**页面三：演唱考试（核心页面）** `/exam`
- 左侧：实时摄像头画面 + 人脸在位状态
- 右侧上方：三张分项分数卡片（音准/节奏/总分）
- 右侧中间：音高实时曲线 Canvas（灰色参考旋律 + 红色学生音高）
- 右侧下方：简谱逐小节滚动高亮（卡拉OK风格）
- 底部：珊瑚粉渐变进度条 + 开始/结束按钮
- 3-2-1 倒计时后自动开始
- **用户体验价值**：边唱边看，即时知道偏了多少

**页面四：考试结果** `/result`
- 总分大卡片（大数字 + 手写体鼓励语）
- 音准/节奏分项卡片（带进度条）
- 错误小节列表（小节号 + 类型 + 说明）
- 改进建议卡片
- 操作按钮：再唱一次 / 换一首
- **用户体验价值**：不只分数，知道哪里要改进

## 二、MVP 核心功能规划

### 2.1 MVP 必做功能及优先级排序

| 优先级 | 功能模块 | 说明 | 当前状态 |
|--------|---------|------|---------|
| **P0** | 考生注册（登录页） | 姓名 + 学号 + 人脸拍照 | ✅ 前端完成，后端接口完成 |
| **P0** | 选曲浏览（选曲页） | 难度筛选 + 曲目列表 + 试听 | ✅ 前端完成，后端接口完成 |
| **P0** | 演唱考试（考试页） | 摄像头 + 录音 + 实时分数 + 音高曲线 + 简谱滚动 | ✅ 前端 UI 完成，评分引擎 stubbed |
| **P0** | 结果展示（结果页） | 总分 + 分项 + 错误 + 建议 + 操作按钮 | ✅ 前端完成 |
| **P0** | 评分引擎 | CREPE 音高 + Librosa 节奏 → 分数 | ⚠️ 后端 stub，待接入真实模型 |
| **P1** | 人脸在位检测 | 考试中定时检测人脸是否在位 | ⚠️ 前端 UI 就绪，后端 stub |
| **P1** | 数据库种子曲目 | ≥3 首曲目的参考数据 | ⚠️ 数据库模型就绪，无种子数据 |
| **P2** | 后端真实启动运行 | FastAPI 服务 + SQLite 部署 | ⚠️ 代码完整，未部署 |
| **P2** | 历史成绩查询 | `/history` 页面 | ⬜ 未开始 |
| **P3** | 分享结果 | 导出图片/链接 | ⬜ 未开始 |

### 2.2 阶段验证目标

| 阶段 | 目标 | 判断标准 |
|------|------|---------|
| **Alpha** | 完整流程可达 | 一个人在电脑前能从"登录→选曲→演唱→看分"走完全程（评分引擎 stub 可接受） |
| **Beta** | 评分准确性 | 真实学生试唱 3 首曲目，系统评分 vs 人工打分误差 ≤ 15% |
| **1.0** | 课堂可用 | 可交付给初中音乐老师做班级摸底测试，10 名学生同时在 3 台设备上使用无异常 |

### 2.3 功能流程与异常处理

**主流程**：

```
登录（人脸+姓名+学号） → 选曲（浏览+试听+确认） → 考试（3-2-1倒计时+演唱+实时评分） → 结果（总分+错误+建议+重练/换曲）
```

**常见异常场景**：

| 异常 | 触发条件 | 产品行为 | 用户感知 |
|------|---------|---------|---------|
| 摄像头无权限 | 浏览器拒绝摄像头请求 | 提示用户手动授权 + 提供手动上传照片代替 | 知道"需要摄像头但可以上传照片" |
| 麦克风无权限 | 浏览器拒绝麦克风请求 | 弹出说明提示，无法进入考试 | 知道"必须允许麦克风才能考试" |
| 人脸丢失 < 5s | 考试中摄像头无人脸 | 无操作，继续录制 | 无感知 |
| 人脸丢失 5-30s | 考试中摄像头无人脸持续 5s+ | 顶部弹出黄色警告条 | 看到"⚠️ 未检测到人脸" |
| 人脸丢失 > 30s | 考试中摄像头无人脸持续 30s+ | 自动暂停考试 + 弹窗确认是否继续 | 考试暂停，需主动点击"继续" |
| 网络中断 | 考试中后端不可达 | 本地缓存录音，结束后自动重试提交 | 如果失败，提示"提交失败请重试" |
| 音高差距过大 | 持续 5s 学生音高偏离参考 > 1 个半音 | 音高曲线区域轻微抖动提示 | 视觉上感觉到"偏多了" |

### 2.4 迭代思路

```
Phase 0 (当前)     Phase 1 (Alpha)     Phase 2 (Beta)      Phase 3 (v1.0)
┌────────────┐    ┌────────────┐     ┌────────────┐      ┌────────────┐
│ 前端 UI 完成 │ →  │ 评分引擎接入 │ →  │ 种子曲目库  │ →   │ 教师端管理  │
│ 后端 stub   │    │ 人脸在位检测 │    │ 历史记录页  │      │ 数据统计    │
│             │    │ 前后端联调  │    │ 分享功能   │      │ 多设备适配  │
└────────────┘    └────────────┘     └────────────┘      └────────────┘
```

### 2.5 1.0 版本目标及后续可拓展方向

**1.0 目标**：
- 可交付学校音乐老师做班级摸底测试
- 曲目库 ≥ 10 首，覆盖三个难度级别
- 支持同时 10 名学生在不同设备上使用

**后续可拓展方向**：
- 教师端管理后台（查看全班成绩、统计正确率分布）
- 练习模式（单句重复练习、慢速跟练）
- 历史趋势曲线（周/月进步曲线）
- 多声部评分（进阶场景）
- 移动端适配（平板模式）
- 第三方登录集成（学校 SSO）
- 导出成绩单 PDF
