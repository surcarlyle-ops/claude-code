# MusicExam 双模式设计文档

## 概述
在现有前端 UI 基础上，将原本单一的"考试"体验拆分为练习模式和考试模式两个独立入口，配套引入频谱进度条可视化、歌词展示、AI 小幽灵陪伴角色，全面提升产品的可用性与体验完整度。

## 设计原则
- **可用性优先** — 可视化与游戏化是辅助，核心是让学生能顺畅完成"选曲->唱->看反馈"流程
- **代码复用** — 练习/考试共用同一页面组件，通过路由参数 mode 区分行为
- **克制新增** — 不引入额外页面或复杂状态管理，在现有架构上做增量改动
- **低依赖** — 所有可视化效果纯前端 Canvas/CSS 实现，不依赖真实音频数据

### 路由参数传递

首页点击入口卡片时，通过 navigation state 传递 mode 参数：

```typescript
navigate('/songs', { state: { mode: 'practice' } })
navigate('/songs', { state: { mode: 'exam' } })
```

选曲页读取 state，在点击曲目卡片时传递给演唱页：

```typescript
navigate('/sing/practice', { state: { songId, mode: 'practice' } })
navigate('/sing/exam', { state: { songId, mode: 'exam' } })
```

### 频谱进度条（山峦填充波浪）

用 Canvas 实现，纯装饰性进度指示，与音频数据无关。

**视觉元素：**
- 基准波浪 — 一条灰色半透明静态正弦波，贯穿进度条宽度
- 已唱填充 — 从起始到当前位置，波浪线以下区域填充玫瑰色渐变（#FB7185 -> #E11D48 渐浅）
- 当前位置 — 一个发光圆点标记，带轻微呼吸光晕动效
- 未唱部分 — 仅显示灰色透明波浪线，无填充色

**行为：**
- 开始时，填充区域为 0
- 演唱中，填充从 0% 推进到 100%（与时间/进度同步）
- 结束后填充定格在 100%，整体转为完成态（可选择渐变呼吸）

### 歌词区

**通用规则：**
- 横向居中、逐行滚动展示（参考 QQ 音乐、网易云标准）
- 字体：大号 Nunito，白色/灰色文字
- 底部滚动：当前行以下透明渐隐
- **重音标记**：在需要重音的字下方标注小符号（✦ 或 ·）
  - 原型阶段：每首歌在第一段歌词中标记 2 处即可
  - 示例：好一朵美丽的✦茉莉花
- **换气标记**：在字与字之间的空隙处标注 💨 符号
  - 原型阶段：每首歌在段落间标记 1-2 处即可
  - 示例：芬芳美丽满枝桠 💨 又香又白人人夸

**模式差异：**

| 特性 | 🎤 练习模式 | 🎯 考试模式 |
|------|------------|------------|
| 歌词显示 | ✅ 常驻 | ⬜ 默认隐藏 |
| 唤出方式 | — | 鼠标悬停/点击或 toggle 按钮 |
| 重音标记 | ✅ | ✅（唤出时同样显示） |
| 换气标记 | ✅ | ✅ |

**考试模式歌词交互：**
- 歌词区域默认显示半透明渐变背景或装饰插画
- 用户点击歌词区域 -> 歌词淡入
- 用户移开鼠标 3 秒后 -> 歌词渐隐
- 或提供一个 toggle 按钮（👁 图标）持久控制显示/隐藏

### AI 小幽灵

**形象描述（占位版本，后续替换）：**
- 灰白色小幽灵，圆溜溜大眼睛，无嘴巴
- 约 48x48px，SVG/CSS 实现
- 固定在页面左下角，浮动定位

**行为逻辑：**

| 阶段 | 行为 | 文案 |
|------|------|------|
| 页面进入 | 从底部弹入，左右晃动 | — |
| 演唱前（练习） | 显示提示泡泡 | "开始前做几个深呼吸，让声音更放松哦~" |
| 演唱前（考试） | 显示提示泡泡 | "别紧张，就像平时练习一样~加油！" |
| 演唱中 | 安静待在角落，轻微上下浮动 | — |
| 演唱中（检测到暂停） | 飘出来提醒 | "别紧张，慢慢来~ 🫶" |
| 练习结束 | 弹跳+鼓励 | "唱得不错呀！注意第二句换气会更棒~ 🔥" |
| 考试结束 | 引导至结果页 | "唱完啦！来看看你的成绩~" |

**提示泡泡样式：**
- 白色圆角气泡
- 小号灰白色文字
- 旁边带小尾巴指向小幽灵

**后续替换：**
- 预留 <img> 组件，src 指向 /src/assets/ghost-*.svg
- 占位阶段使用纯 CSS/SVG 内联绘制

## 结果页（ResultPage）改动

### 模式差异

| 元素 | 🎤 练习模式 | 🎯 考试模式 |
|------|------------|------------|
| AI 小幽灵开场 | "唱完啦！听听小幽灵想说什么~" | "成绩出来啦！" |
| 总分展示 | 不出等级，由小幽灵口吻评价 | 完整等级制（A+ / A / B+ / B / C / D） |
| 等级颜色 | 不适用 | 现有（A+ 金 / A 薄荷 / B+ 天蓝 / B 紫 / C 黄 / D 灰） |

| 底部按钮 | "再练一次" / "换一首" | "再考一次" / "换一首" / "查看记录" |

### 练习模式结果页样式
- 整体色调偏温暖（淡粉/暖黄背景）
- AI 小幽灵居中或放大展示
- 评价文案示例：
  - "很棒！你已经唱得很好了~ 🌟"
  - "第二句换气可以更顺畅一些，下次试试！"
  - "整体节奏感很棒，音准再练练就更厉害了！"
- 按钮文案偏轻松活泼

## 歌词数据

### 格式

```typescript
interface LyricLine {
  time: number     // 秒，从歌曲开始
  text: string     // 歌词文本
  accent?: number[]    // 重音字符索引数组，如 [2, 5] 表示第3、6个字
  breath?: boolean     // 该行结束后是否需要换气
}
```

### 原型阶段
所有 14 首曲目使用同一套歌词数据结构，每首歌在第一段歌词中标记 2 处重音+1-2 处换气即可。
