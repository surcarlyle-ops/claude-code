# UI 参考图接入指南

## 用途

当用户投喂 UI 参考图（截图、设计稿、竞品界面等）时，按本流程操作。

---

## 流程

### 第一步：接收参考图

识别图片中的可借鉴元素：

| 维度 | 具体看什么 |
|------|-----------|
| 配色 | 主色、辅色、背景色、文字色、渐变 |
| 字体 | 衬线/无衬线、字重、字距、字号层级 |
| 组件 | 按钮、卡片、输入框、导航栏、弹窗、进度条 |
| 布局 | 间距、对齐方式、层次结构、栅格 |
| 交互 | 悬停态、动效、反馈方式、过渡 |

### 第二步：对比分析

阅读 [CLAUDE.md](../CLAUDE.md) 了解项目当前设计风格：

- 玫瑰粉主色调（`#E11D48` / `#FB7185`）
- 马卡龙色系卡片、暖白背景（`#FFF1F2` / `#FFFAF5`）
- 大圆角体系（卡片 20px、按钮 16px、输入框 14px）
- 字体 Nunito + Caveat
- GSAP 动效（渐入、stagger、悬浮上移）
- 桌面优先，刚刚完成自适应布局改造（container-wide + text-responsive-* 系列）

找出参考图与现有风格的**可融合点**和**冲突点**。

### 第三步：提方案

每次看图后，先问用户：

> "这张图里你最喜欢的是什么？配色/组件/整体氛围/具体某个元素？"

确认用户想**全盘替换**还是**局部借鉴**，再动手。

### 第四步：执行更新

1. 用户确认方案后，修改对应页面/组件的 CSS
2. 保持项目现有设计规范（大圆角、暖白背景、活泼风格）
3. 响应式类（container-wide、text-responsive-*）不要改动
4. 改完后构建验证：`npx vite build`
5. 然后 `git commit && git push`
