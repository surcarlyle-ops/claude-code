# MusicExam 产品设计规范（PRD + Design Spec）

> 版本：v1.0 | 日期：2026-06-09 | 状态：待开发

---

## 一、产品定义

### 1.1 产品名称建议
**MusicExam** — 面向初高中生的音乐考试模拟练习平台

### 1.2 核心定位
为初高中生提供"对着屏幕唱、即时看分数"的音乐自学评分工具。把开源 AI 音高模型 + 人脸身份验证 + 极简 Web 交互，做成门槛最低的自评系统。

### 1.3 核心价值
| 用户痛点 | 产品解法 |
|----------|----------|
| 自己唱歌听不出音准偏差 | 实时音高曲线 vs 参考旋律对比 |
| 节奏不稳但没人指正 | Librosa 节拍检测 + 节奏偏差可视化 |
| 没有低门槛的音乐自学工具 | 浏览器打开即用，无需安装 App |
| 缺乏即时的量化反馈 | 演唱中实时显示分项分数 |
| 想自己练但不知道从哪里开始 | 分级曲目库 + 改进建议 |
| 音乐课考试前没有模拟练习途径 | 完整模拟考试流程（登录→选曲→演唱→结果） |

### 1.4 核心用户画像
- **年龄段**：12–18 岁，初高中在校学生
- **使用场景**：在家使用电脑（带摄像头 + 麦克风），打开浏览器练习
- **技术基础**：会用手机/电脑基础功能，无需音乐基础
- **动机**：音乐课考试备考、自学兴趣、老师布置的练习任务
- **频率**：每周 1–3 次，每次 10–20 分钟

### 1.5 差异化
- **全开源模型栈**：CREPE + Librosa，无 API 调用成本，支持离线部署
- **即开即用**：浏览器打开 → 选曲 → 开唱 → 看分，无冗余步骤
- **人脸防替考**：摄像头拍照绑定身份 + 全程在位检测
- **治愈系 UI**：低 AI 味、温暖柔和，区别于冰冷的技术工具

## 二、设计体系（Design System）

### 2.1 设计价值观
- **治愈感优先**：让学生面对的是一个"温柔的工具"，而不是一个"考官"
- **即时反馈**：唱的每一句都能看到音高位置，减少焦虑
- **零学习成本**：界面元素自解释，不给学生增加认知负担

### 2.2 配色方案

| 角色 | 色值 | Tailwind 映射 | 用途 |
|------|------|--------------|------|
| 主背景 | `#FFFAF5` | 自定义 `bg-surface` | 页面底色 |
| 卡片背景 | `#FFFFFF` | `bg-white` | 所有卡片 |
| 浅粉（主色） | `#FADADD` | 自定义 `bg-pink-soft` | 头像底色、进度卡片 |
| 奶油黄 | `#FDF4D0` | 自定义 `bg-yellow-cream` | 通知卡片、高亮提示 |
| 浅天蓝 | `#D6ECFB` | 自定义 `bg-blue-sky` | 音准相关卡片 / 图表 |
| 浅薄荷绿 | `#D5F5E3` | 自定义 `bg-mint` | 节奏相关卡片 / 通过状态 |
| 淡紫 | `#E8DAEF` | 自定义 `bg-purple-lavender` | 选曲卡片、装饰元素 |
| 珊瑚粉（点缀） | `#FF6B6B` | 自定义 `text-coral` | 错误标记、重要按钮 |
| 青蓝（点缀） | `#4ECDC4` | 自定义 `text-teal-mint` | 正确标记、成功状态 |
| 正文色 | `#2D3436` | `text-gray-800` | 主要文字 |
| 辅助文字 | `#636E72` | `text-gray-500` | 说明文字 |
| 导航栏底色 | `#2D3436` | `text-gray-800` 底色 | 底部导航 |

**配色原则**：
- 同一页面不超过 3 种主彩色卡片，其余用白色留白
- 点缀色（珊瑚粉 / 青蓝）仅用于功能性高亮（分数、错误标记、CTA 按钮），不用于大面积装饰
- 对比度确保可读性，正文与背景至少 4.5:1

### 2.3 字体系统

| 层级 | 字体 | 字号 | 字重 | 用途 |
|------|------|------|------|------|
| 页面标题 | "Nunito" / 系统无衬线 | 28px | Bold (700) | 每页顶部主标题 |
| 区块标题 | "Nunito" / 系统无衬线 | 20px | SemiBold (600) | 卡片标题、分区标题 |
| 正文 | "Nunito" / 系统无衬线 | 16px | Regular (400) | 一般内容文字 |
| 辅助文字 | "Nunito" / 系统无衬线 | 14px | Regular (400) | 说明、标签、日期 |
| 大数字 | "Nunito" / 系统无衬线 | 36–48px | Bold (700) | 分数、统计数据 |
| 手写点缀 | "Caveat" 或类似 | 随场景 | Regular | 仅用于问候语、鼓励提示，不超过全文 5% |

**字体原则**：
- 主字体选圆润无衬线体（Nunito 为首选，系统 fallback）
- 不使用直角/尖锐字体
- 手写体仅做情感点缀，不做功能文字
- 中英文混排时用系统默认中文字体

### 2.4 圆角与间距

| 元素 | 圆角 | 说明 |
|------|------|------|
| 大卡片（功能卡片、曲目卡片） | 20–24px | 视觉上的"软" |
| 小卡片（数据卡片、标签） | 16px | 与大卡片统一风格 |
| 按钮 | 16–20px | 圆润感 |
| 输入框 | 12–16px | 略硬一点便于识别可输入 |
| 进度条 | 9999px（全圆角） | 拉满圆形效果 |
| 头像 | 50%（正圆） | 标准圆形头像 |
| 图标按钮（圆形功能入口） | 50%（正圆） | 首页快捷功能区 |

| 区域 | 间距 | 说明 |
|------|------|------|
| 卡片之间 | 16–20px | 充足留白 |
| 卡片内边距 | 20–24px | 内部不拥挤 |
| 页面左右边距 | 20px | 统一 |
| 区块之间 | 24–32px | 视觉分段 |

### 2.5 阴影
- **默认卡片**：`box-shadow: 0 2px 12px rgba(0,0,0,0.06)` — 非常轻的阴影，几乎不可见
- **悬停卡片**：`box-shadow: 0 4px 16px rgba(0,0,0,0.10)` — 轻微的抬高感
- **不使用**：重阴影、多层阴影、深色投影（破坏"轻"的感觉）

### 2.6 图标与装饰
- **图标风格**：极简线性图标，2px 描边，圆角端点
- **推荐图标集**：Phosphor Icons（风格统一、有 fill/duotone 变体）
- **点缀元素**：小星星（★）、圆点（●）、对话气泡、小花朵 — 仅用于空状态或鼓励提示
- **不使用**：复杂拟物图标、3D 图标、过多装饰元素

### 2.7 动画原则
- **过渡**：`transition-all duration-300 ease-in-out` — 柔和、不快
- **hover**：卡片轻微上浮（`translateY(-2px)`）+ 阴影加深
- **加载**：圆角进度条动画，或简单的呼吸动画
- **页面切换**：fade-in，250ms
- **不使用**：弹跳动画、旋转动画、过于戏剧化的动效

### 页面 1：登录 / 考生身份确认

**页面路径**：`/login`

**页面目标**：完成考生身份登记（姓名 + 学号 + 人脸照片绑定），为后续考试确保成绩真实对应个人。

│  └────────────────────────────────┘  │
│                                      │
│  ┌────────────────────────────────┐  │
│  │  姓名                            │  │ ← 白色输入卡片

│                                      │
│  ┌────────────────────────────────┐  │

│  │  ┌──────────────────────────┐   │  │

│  └────────────────────────────────┘  │
│                                      │

│  首次使用？看看怎么操作 →              │ ← 轻文字链接

```

#### 交互逻辑
1. **人脸拍照**：点击圆形头像区 → 浏览器请求摄像头权限 → 显示实时画面 + 拍照按钮 → 点击拍照 → 预览 + 重拍/确认
2. **防替考机制**：考试过程中定时（每 30s）后台截帧与注册照片做人脸匹配（MediaPipe Face Mesh），不匹配则标记警告
3. **登录按钮**：姓名 + 学号 + 人脸照片三者齐全才可点击（否则置灰）
4. **跳过人脸（管理员模式）**：可配置跳过人脸（用于测试/演示），但非默认行为

#### 状态与异常
| 状态 | UI 反馈 |
|------|---------|
| 摄像头无权限 | 提示"请在浏览器设置中允许摄像头访问" + 手动上传照片替代 |
| 摄像头打开中 | 头像区显示 loading 动画 |
| 姓名为空 | 输入框边框变珊瑚粉 + 底部提示"姓名不能为空" |
| 学号重复 | 提示"该学号已注册，请确认是否是你本人" |
| 提交成功 | 淡入过渡到选曲页 |

interface Song {

```

### 页面 3：演唱考试（核心页面）

**页面路径**：`/exam`

**页面目标**：学生对着摄像头和麦克风演唱指定曲目，系统实时显示音准对比和分数。

#### 布局结构（三栏式）

```
┌──────────────────────────────────────────────────┐
│  ← 退出    《小星星》考试中    00:48 / 1:20       │ ← 顶部栏（紧凑）
│                                                  │
│  ┌────────┐  ┌─────────────────┐  ┌───────────┐  │

│  │ 摄像头 │  │  音高实时曲线    │  │ 🟢━━━━━━  │  │   三张小卡片
│  │ 画面   │  │                 │  │          │  │   浅天蓝（音准）
│  │        │  │  🟢 你的演唱    │  │ 节奏  85%  │  │   浅薄荷绿（节奏）
│  │ (人脸) │  │  ⚪ 参考旋律    │  │ 🟢━━━━━━  │  │   淡紫（总分）
│  │        │  │                 │  │          │  │
│  │在位 ●  │  │                 │  │ 总分  89  │  │
│  └────────┘  └─────────────────┘  └───────────┘  │
│                                                  │
│  ┌──────────────────────────────────────────────┐ │
│  │  ♩ ♩ ♩ ♩ | ♩ ♩ ♩ ♩ | ♩ ♩ ♩ ♩ | ♩. ♪ ♩ ♩ │ │ ← 简谱滚动区
│  │         ↑ 当前小节（高亮）                    │ │   卡拉OK风格
│  └──────────────────────────────────────────────┘ │
│                                                  │
│  [████████████░░░░░░░░░] 65%                     │ ← 全圆角进度条
│                                                  │
│  [结束演唱]                                       │ ← 珊瑚粉按钮
│                                                  │
│  [████████░░] 底部导航                            │
└──────────────────────────────────────────────────┘
```

#### 交互逻辑
1. **倒计时**：进入页面后 3–2–1 倒计时，然后自动开始录音 + 评分
2. **摄像头**：左侧持续显示，背景做 MediaPipe 人脸在位检测
3. **音高曲线**：Canvas 实时绘制——绿色曲线 = 学生音高，灰色虚线 = 参考旋律；X 轴时间、Y 轴音高
4. **分项分数**：每 2 秒更新一次百分比（后端流式返回或前端本地对比）
5. **简谱滚动**：随着 MIDI 播放进度，当前小节高亮 + 自动滚动
6. **进度条**：灰色底色 + 珊瑚粉渐变填充，全圆角
7. **结束演唱**：可提前结束（学生点击），或曲目播完自动结束
8. **人脸在位检测**：如果摄像头中无人脸超过 5 秒 → 顶部弹出警告"⚠️ 未检测到人脸，请保持正对屏幕"

#### 状态与异常
| 状态 | UI 反馈 |
|------|---------|
| 麦克风无权限 | 弹出提示"需要麦克风权限才能评分"，不进入考试 |
| 3-2-1 倒计时 | 屏幕中央大数字倒计时 + 呼吸动画 |
| 人脸丢失 > 5s | 顶部 toast 警告条 |
| 人脸丢失 > 30s | 自动暂停考试，弹出"是否继续？" |
| 音高差距过大持续 5s | 音准曲线区域轻微抖动提示 |
| 网络中断（如有后端） | 切换本地降级评分，结束后同步 |
| 考试结束 | 自动过渡到结果页 |

#### 交互逻辑
1. **进入动画**：总分数字从小到大滚动到位（数字滚动动画）
2. **错误小节**：点击某一错误 → 展开显示该小节的音高曲线局部放大图
3. **再唱一次**：直接重新进入同一曲目的考试页
4. **换一首**：跳转回选曲页
5. **查看历史**：跳转到历史记录页（MVP 阶段可为页面 4 的扩展，或单独的 `/history` 页）

## 四、交互流程

### 4.1 主流程
```
登录/注册（人脸拍照） → 选曲（浏览曲目 + 试听） → 考试（演唱 + 实时评分） → 结果（评分 + 建议 + 重练）
```

### 4.2 状态转换图
```
[未登录] ──完成人脸+姓名+学号──→ [已登录]
[已登录] ──选中曲目─────────→ [准备考试]
[准备考试] ──3-2-1倒计时─────→ [考试中]
[考试中] ──曲目结束/手动结束─→ [评分中]
[评分中] ──生成结果──────────→ [查看结果]
[查看结果] ──再唱一次────────→ [准备考试]
[查看结果] ──换一首─────────→ [已登录]
```

### 4.3 异常路径
- 摄像头无权限 → 允许手动上传照片 → 考试中关闭在位检测（降级）
- 麦克风无权限 → 无法进入考试页，提示授权
- 人脸丢失 → 警告 → 超时暂停
- 浏览器不支持 Web Audio → 提示使用 Chrome/Edge

## 五、MVP 功能规划

### 5.1 优先级排序

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 登录 + 人脸拍照 + 身份绑定 | 基础流程起点 |
| P0 | 选曲页 + 曲目库（≥3 首） | 核心流程 |
| P0 | 考试页（录音 + 摄像头 + 音高曲线 + 实时评分） | 产品核心价值 |
| P0 | 结果页（总分 + 分项 + 错误标记 + 建议） | 闭环反馈 |
| P0 | CREPE 音高评分引擎 | 核心算法 |
| P0 | Librosa 节奏评分引擎 | 核心算法 |
| P1 | 简谱滚动显示 | 提升演唱体验 |
| P1 | 考试中人脸在位检测 | 防作弊 |
| P1 | 曲目试听播放 | 选曲体验 |
| P2 | 历史记录页 | 查看过往成绩 |
| P2 | 分享结果 | 社交传播 |
| P3 | 多设备适配（平板） | 更多场景 |

### 5.2 阶段验证目标
- **Alpha**（P0 完成）：一个人能从头到尾走完"登录 → 选曲 → 唱 → 看分"
- **Beta**（P1 完成）：真实学生试唱 3 首曲目，对比人工打分 vs 系统评分误差 ≤ 15%
- **1.0**（P2 完成）：可交付给学校音乐老师做班级测试

## 六、不做什么（边界）

- ❌ 和弦 / 调性分析（初级不需要，技术复杂度太高）
- ❌ 情感识别 / 表现力评分（初级不讲这个）
- ❌ 音色评估 / 嗓音评价（超出初级评分范围）
- ❌ 复杂账号系统（注册/找回密码/多设备同步 — MVP 不做）
- ❌ 教师端管理后台（1.0 不做）
- ❌ 班级管理 / 批量导入（1.0 不做）
- ❌ 多声部 / 合唱评分（仅独唱）
- ❌ 移动端适配（MVP 仅桌面端）
- ❌ 实时流式评分（MVP 用本地计算 + 简单后端即可）
- ❌ 高分辨率人脸识别（Haar Cascade / MediaPipe 足以区分"有人 vs 没人"）
