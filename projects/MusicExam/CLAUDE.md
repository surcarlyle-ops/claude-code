# CLAUDE.md — MusicExam 项目

## 项目简介
面向初高中生的音乐初级考试评分系统。学生对着摄像头唱指定曲目，系统通过开源模型做音准+节奏评分，实时反馈结果。

⏳ 开发落地

---

## 核心需求
1. **音乐评分**：基于开源模型（音准 + 节奏），初高中生基础评分
2. **交互系统**：学生对着系统唱歌，实时评分反馈
3. **人脸捕捉**：摄像头对接，拍到人脸并对应考生身份

## 技术方案（已确定：方案 A）

### 技术栈
| 层 | 技术 |
|---|---|
| 前端 | React + TypeScript + Vite + Tailwind CSS |
| 后端 | Python FastAPI + Uvicorn |
| 音高模型 | CREPE（MIT License）— 行业 SOTA |
| 节奏分析 | Librosa（ISC License）— 节拍/节奏评估 |
| 人脸检测 | OpenCV Haar Cascade / MediaPipe |
| 数据库 | SQLite（MVP）→ 后续可升级 PostgreSQL |

### 模型调研清单
| 模型 | License | 用途 | 选择理由 |
|------|---------|------|----------|
| CREPE ⭐ | MIT | 音高追踪 | 行业 SOTA 精度，pip install crepe |
| Librosa ⭐ | ISC | 节奏分析 | 业界标准，beat/onset 检测成熟 |
| OpenCV ⭐ | Apache 2.0 | 人脸检测 + 摄像头 | 强制需求 |
| Essentia | AGPL v3 | 综合备选 | 内置 CREPE + TempoCNN，可做降级 |
| Basic Pitch | Apache 2.0 | 多声部转录（备选） | Spotify 出品，单声道场景用不上 |

### 架构概览
```
浏览器 (React App)
├── 摄像头模块（人脸确认 + 全程在位检测）
├── 录音模块（Web Audio API）
└── 实时结果展示（音准曲线 + 节奏对比 + 分数）
        │
        ▼ HTTP
Python 后端 (FastAPI)
├── 评分引擎 (Scoring Engine)
│   ├── CREPE → 音高提取 → 与参考旋律对比 → 音准分
│   ├── Librosa → 节拍检测 → 与参考节奏对比 → 节奏分
│   └── 加权平均 → 综合分 + 分项分
├── 曲目管理（参考曲目 / MIDI / 乐谱）
├── 成绩管理（存储查询 / 数据分析）
├── 考生管理（人脸注册 / 身份验证）
└── SQLite 数据库
```

## 页面流程（已确定 4 页）

### 页面 1：登录 / 考生身份确认
- 学生输入姓名、学号
- 开启摄像头 → 拍人脸 → 后台保存人脸截图
- 身份绑定：姓名 + 学号 + 人脸图像对应
- **用户价值**：确保考试结果真实对应个人，防替考

### 页面 2：选择考试曲目
- 曲目列表按难度分级（初级/中级）
- 每首曲目显示简谱预览，可试听参考旋律
- **用户价值**：看得见考题，提前心里有数

### 页面 3：演唱考试（核心页面）
- 左侧：实时摄像头画面（人脸确认持续在位）
- 右侧：音准动态曲线（绿色=学生演唱，灰色=参考旋律）
- 顶部：进度条 + 实时分项分数（音准%、节奏%）
- 底部：简谱滚动显示（类似卡拉OK歌词滚动）
- **用户价值**：唱的过程中就能看到自己音准偏了多少，节奏对不对

### 页面 4：考试结果
- 总分 + 分项分（音准/节奏）
- 错误标记：具体哪几个小节有问题
- 建议：针对弱项给出改进方向
- **用户价值**：不只是分数，还知道哪里唱得好、哪里需改进

## 产品定义（Prompt 模板 1 框架）

- **产品类型**：教育工具类
- **核心定位**：为初高中生提供自学自练场景下的音乐演唱评分反馈工具
- **目标用户**：初高中学生（12-18岁），不同年级可选不同难度曲目
- **使用场景**：学生在家模拟练习，对着系统唱指定曲目
- **核心用户痛点**：
  1. 自己唱歌听不出音准偏差
  2. 节奏不稳但没人指正
  3. 没有低门槛的音乐自学工具
  4. 缺乏即时的量化反馈
  5. 想自己练但不知道从哪里开始
  6. 音乐课考试前没有模拟练习途径
- **差异化**：把开源 AI 音高模型 + 人脸身份验证 + 简单 Web 交互，做成低门槛的自评工具
- **不做什么**：
  - ❌ 不做和弦/调性分析（初级不需要）
  - ❌ 不做情感识别（超出初级范围）
  - ❌ 不做音色评估（初级考试不评分）
  - ❌ 不做复杂账号系统（MVP 阶段）
