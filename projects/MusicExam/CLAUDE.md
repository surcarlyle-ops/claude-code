# CLAUDE.md — MusicExam 项目

## 项目简介
面向初高中生的音乐考试练习平台。学生对着摄像头唱指定曲目，系统通过 AI 模型做音准+节奏评分，实时反馈结果。目标是在家里/学校的电脑上打开浏览器就能练。

## 调研结论（2026-06-10 深度调研）

### 中小学声乐考试的真实情况
- **考试形式**：通常是清唱一首歌，老师主观打分
- **现实困境**：一个班 50-60 人，每人平均只有 30 秒评分时间
- **评分标准**：高度主观，缺乏标准化工具
- **课时**：每周仅 1 节音乐课，经常被主科挤占

### 学生核心痛点
1. **音准和节奏是最难的部分** — 武汉 AI 试点中学生的真实反馈
2. **起调问题** — 儿童声带发育中，音域不同，起调太高/太低整首歌唱不好
3. **缺乏个体反馈** — 考试只给一个分，不知道哪里错了怎么改
4. **考试焦虑** — 当众清唱容易紧张发挥失常

### 已有解决方案
- 「音乐星球」微信小程序（武汉试点，千余学生）
- 「行者AI美育」商用方案（学校采购型）
- 学术界：双维评价框架（AI 评分量化维度 + 教师保留质性判断）

### 产品定位
**"让学生自己对着电脑唱、即时看分数、知道哪里错了"**
- 不是手机端场景，而是学生在家/学校用电脑 + 浏览器 + 摄像头/麦克风
- 核心功能聚焦：选歌 → 唱 → 看分，不做多余功能

## 6 个页面一览

### 0. 首页 `/`（新增）
- Hero 区：大 Logo + 副标题 +「开始练习」按钮
- 最近练习记录列表（带分数和日期）
-「查看趋势」入口
- 无多余功能模块

### 1. 登录页 `/login`
- **左右分栏布局**（桌面端）：左 = 品牌展示 / 右 = 表单
- 姓名 + 学号输入 + 摄像头拍照绑定人脸
- 底部「跳过登录，直接体验」快速进入首页

### 2. 选曲页 `/songs`
- 顶部固定栏（返回 + 标题 + 头像）
- 难度筛选标签（初/中/高级）
- **响应式卡片网格**（1-2-3 列自适应桌面宽屏）
- 曲目卡片：🎵 图标 + 标题 + 难度 + 时长 + 试听按钮 + 开始演唱按钮
- **点击直接进入考试**（去掉确认弹窗，减少步骤）

### 3. 考试页 `/exam`（核心）
- **左右两栏布局**（桌面宽屏）
- 左栏：摄像头画面 + 人脸在位状态
- 右栏上方：三张分项分数卡（音准=天蓝/节奏=薄荷绿/总分=淡紫，渐变底色）
- 右栏中间：Canvas 音高曲线（灰色虚线=参考旋律，红色实线=学生音高）
- 右栏下方：简谱逐小节滚动高亮
- 底部：珊瑚粉渐变进度条 + 开始/结束按钮
- 3-2-1 倒计时后自动开始

### 4. 结果页 `/result`
- **两列布局**：左列 = 总分大卡 + 操作按钮 / 右列 = 分项+错误+建议
- 总分数字滚动动画
- 音准/节奏分项卡片（带进度条）
- 错误小节列表（小节号 + 类型 + 说明）
- 改进建议卡片
- 操作按钮：再唱一次 / 换一首 / 查看趋势

### 5. 趋势页 `/trends`（新增）
- 时间筛选（7天 / 30天 / 自定义）
- Canvas 折线图：得分趋势，柔和的珊瑚粉渐变填充
- 练习记录列表
- 右侧摘要卡片：平均分 / 最高分 / 练习次数

## 核心交互流程

```
首页 → 登录（姓名+学号+人脸） → 选曲（筛选+选择） → 考试（唱+实时评分） → 结果（分数+错误+建议）
  ↑                                                                         ↓
  └────────────────── 首页显示最近记录 ────────────────────── 再唱一次/换一首 ──┘
```

## 设计原则

1. **桌面优先** — 页面宽度使用 `max-w-5xl`，充分利用宽屏
2. **极简功能** — 只保留"选歌→唱→看分"核心流程，去掉了待办任务、快捷功能等多余模块
3. **马卡龙治愈系** — 低饱和粉/黄/蓝/绿/紫，大圆角卡片(16-24px)，充足留白
4. **渐变底色** — 卡片使用 `bg-gradient-to-br` 渐变，增加层次感
5. **微动效** — 总分滚动动画、卡片悬浮上移、按钮过渡

## 当前已知限制

- 考试页分数是 `Math.random()` 模拟，非真实评分
- 音高曲线数据是正弦波模拟，非真实音频分析
- 提交评分会调后端 API，但后端返回硬编码结果
- `faceLost` 状态已就位但无人脸检测逻辑触发
- 数据库目前是空的（无种子曲目数据）
- 后端 API 存在安全风险（IDOR 等），待后续开发阶段处理
